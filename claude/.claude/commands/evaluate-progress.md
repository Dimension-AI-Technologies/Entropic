name: evaluate-progress
description: Objectively evaluate task completion and quality
---

Evaluate the current state of: $ARGUMENTS

Provide:
1. **Completion Status**: What percentage is actually done vs claimed
2. **Quality Assessment**: Code quality, documentation, testing coverage
3. **Remaining Work**: Specific tasks still needed
4. **Risk Assessment**: What could go wrong or break
5. **Improvement Areas**: Concrete suggestions for enhancement

Use factual language. Avoid words like "perfect," "excellent," "amazing."
